# Implementation Plan: Module 1 — The Robotic Nervous System (ROS 2)

**Branch**: `001-ros2-nervous-system` | **Date**: 2026-06-13 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-ros2-nervous-system/spec.md`

## Summary

Produce four MDX chapter files for Docusaurus (Module 1: ROS 2) that teach the publish-subscribe
architecture using the "Nervous System" analogy, with zero mathematical derivations. Integrate a
context-scoped RAG chatbot widget on every page and update the Docusaurus sidebar to link all
four chapters under a Module 1 category.

**Language/Version**: MDX 3.x (Docusaurus v3) + React 18 (chat widget stub)

## Phase 0: Research Findings

*See [research.md](./research.md) for full detail. Key decisions:*

- **MDX component integration**: Global registration in `docusaurus.config.js` preferred over
  per-file imports. If not yet configured, a per-file import fallback is documented in quickstart.
- **Sidebar format**: Manual `sidebars.js` category definition chosen over autogenerated to
  guarantee chapter ordering (ch01 → ch04).
- **RAG chunking boundary**: `##` H2 headings confirmed as the correct split boundary.
  Minimum 4 H2 sections per chapter (≥3 content chunks after excluding metadata sections).
- **RagChatbot stub**: Zero-dependency stub at `src/components/RagChatbot.jsx` prevents build
  failures while the full backend widget is under development.
- **Word count gate**: ≤3,500 prose words per chapter (≤4,500 total including code blocks).

## Phase 1: Design

### Content Architecture

Each chapter follows a rigid internal structure to maximise RAG chunk quality and reading clarity:

```text
---
title: "Chapter N — [Title]"
---

# Chapter N — [Title]                      ← exactly one H1

## Learning Objectives                     ← MUST be first H2
- 3–5 testable outcome bullets

## [Section A — Self-Describing Heading]   ← content H2 (topic-qualified)
prose + :::admonition (at least one per file)

## [Section B — Self-Describing Heading]  ← content H2
prose + optional tagged code block

## [Section C — Self-Describing Heading]  ← content H2 (optional additional)

## Summary                                 ← MUST be last H2
3–5 sentence recap

<RagChatbot context="module-1" />         ← after all prose, end of file
```
